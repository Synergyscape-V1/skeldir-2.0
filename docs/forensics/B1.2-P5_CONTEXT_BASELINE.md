# B1.2-P5 Context Baseline

Scope: hypothesis validation for B1.2-P5 revocation substrate (denylist + `tokens_invalid_before`) against authority code and CI.

## Hypothesis Matrix (H01-H06)

| ID | Verdict | Evidence |
|---|---|---|
| H01: denylist table missing or not tenant-scoped | Validated (pre-remediation) | No denylist substrate existed in `main` before this remediation pass; no `auth_access_token_denylist` migration/model/service references were present. |
| H02: `tokens_invalid_before` substrate absent | Validated (pre-remediation) | No `auth_user_token_cutoffs` substrate existed in migrations/models/services before this remediation pass. |
| H03: access JWT missing revocation primitives (`jti`,`iat`) | Partially validated (pre-remediation) | Existing tests and runtime tokens had `iat`; `jti` was not required/validated in the verifier. |
| H04: verifier enforcement fragmentation across API + worker | Validated (pre-remediation) | API used `Depends(get_auth_context)` but worker envelope path had no revocation check parity. |
| H05: revocation implementation could violate bounded compute | Open risk (pre-remediation) | No revocation substrate meant no index-backed bounded-lookup guarantees for denylist/kill-switch checks. |
| H06: CI lacked non-vacuous revocation proofs | Validated (pre-remediation) | Existing negative controls had no explicit denylist/kill-switch bypass proofs. |

## Verification Entry Points (canonical)

- HTTP dependency verifier: `get_auth_context` in `backend/app/security/auth.py`
- Worker token decode + revocation entrypoint: `assert_worker_auth_envelope_active` in `backend/app/tasks/context.py`
- Route boundary confirmation:
  - JWT routes are dependency-driven in `backend/app/main.py` through bearer-auth route families.
  - Webhook/HMAC routes remain separate in `backend/app/api/webhooks.py` and `backend/app/webhooks/signatures.py`.

## CI Enforcement Plane (owner jobs)

- Runtime/contract topology gate: `tests/contract/test_contract_semantics.py`
- Non-vacuous negative controls gate: `scripts/contracts/run_negative_controls.sh`
- Revocation substrate runtime proofs: `backend/tests/test_b12_p5_revocation_substrate.py`
