# B1.2-P5 Remediation Evidence

## Summary

B1.2-P5 now enforces immediate access-token revocation via:

- `jti` denylist enforced in canonical auth verification dependency.
- `tokens_invalid_before` kill-switch (`tenant_id` + `user_id`) enforced against JWT `iat`.
- Logout denylisting current token.
- Admin kill-switch endpoint: `POST /api/auth/admin/token-cutoff`.
- Worker auth-envelope revocation parity before side effects.
