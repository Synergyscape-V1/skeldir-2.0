## Initial Forensic Findings (Validated)
1. R1 runtime boot failure from missing dependency in CI install path.
   - Evidence: `R1: Contract/Runtime Viability` run `22697231346` failed with `ModuleNotFoundError: No module named 'bcrypt'`.
   - Root cause: `backend/requirements-lock.txt` was missing `bcrypt` while runtime imports `bcrypt` in `backend/app/services/auth_tokens.py`.
2. R1 closure gate false negative on login "happy path".
   - Evidence: `R1` run `22698025896` returned `422` on "valid request".
   - Root cause: workflow payload omitted required `tenant_id` for `LoginRequest`.
3. R1 schema idempotency false drift signal.
   - Evidence: same run `22698025896` reported Run1 != Run2 with diff only in `\restrict` / `\unrestrict` tokens from `pg_dump`.
   - Root cause: hash comparison used raw `pg_dump` output including volatile session tokens.

## Remediation Applied
1. PR `#167` (merge commit `48ae423a9947b541ddae2eaa45a4c0ab684abc7e`)
   - Change: add `bcrypt==4.1.3` to `backend/requirements-lock.txt`.
2. PR `#168` (merge commit `0e9997c32d9be2801a2fb7cbd8e59009753ad215`)
   - Change: normalize schema dumps before hash comparison in `.github/workflows/r1-contract-runtime.yml` by removing volatile `\restrict`/`\unrestrict` lines.
   - Change: set `CONTRACT_TESTING=1` for R1 live stack boot.
   - Change: include required `tenant_id` in R1 login happy-path payload.
