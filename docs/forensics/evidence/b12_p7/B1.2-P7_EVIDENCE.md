# B1.2-P7 Evidence Pack

## Initial Findings (Validated Baseline)

1. Tenant-scoped worker execution authority was heterogeneous.
- Some tenant tasks depended on ad-hoc kwargs and decorator behavior instead of a universal worker-plane contract.

2. Producer-side tenant enqueue enforcement was not globally chokepointed.
- Tenant task dispatch occurred through mixed entrypoints and Celery primitives, creating bypass risk.

3. Revocation semantics were not encoded in a first-class worker envelope.
- Session revocation checks existed, but session claims were not uniformly propagated as a required tenant-task contract.

4. Non-session producers (webhooks/beat fanout) had legitimate tenant work without session keys.
- This required explicit system authority semantics rather than synthetic user claims.

5. Global-task non-write guarantees were not backed by a non-vacuous denial proof.

## Remediations Implemented

1. Polymorphic authority envelope introduced.
- `SessionContext`: `tenant_id`, `user_id`, `jti`, `iat`.
- `SystemContext`: `tenant_id` only.
- Files:
  - `backend/app/tasks/authority.py`

2. Canonical tenant enqueue choke-point added and adopted.
- `enqueue_tenant_task(...)` for direct enqueue.
- `tenant_task_signature(...)` for Celery canvas/chains.
- Files:
  - `backend/app/tasks/enqueue.py`
  - `backend/app/api/webhooks.py`
  - `backend/app/services/attribution.py`
  - `backend/app/services/llm_dispatch.py`
  - `backend/app/tasks/matviews.py`
  - `backend/app/tasks/maintenance.py`

3. Worker-side default-deny enforced via custom Celery base task.
- `TenantTask` requires `authority_envelope` (strict mode enforced in CI/runtime proofs).
- Session envelopes execute revocation checks before task side effects.
- System envelopes bypass user revocation safely while preserving tenant binding.
- Files:
  - `backend/app/tasks/tenant_base.py`
  - `backend/app/tasks/context.py`

4. Tenant binding parity preserved before SQL in tenant tasks.
- Tenant-scoped tasks moved to `base=TenantTask`.
- Existing transaction-local GUC path retained via session helpers and worker pre-run binding.

5. Beat and webhook tenant fanout converted to `SystemContext`.
- No synthetic `jti/iat` introduced for system-origin authority.
- New all-tenant wrappers added for maintenance schedules.

6. Non-vacuous global-task write-denial proof added.
- SQL interception guard plus deliberate DML negative control.
