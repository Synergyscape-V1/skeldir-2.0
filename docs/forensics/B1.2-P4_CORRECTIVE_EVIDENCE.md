# B1.2-P4 Corrective Evidence

Date: 2026-03-03  
Scope: Corrective remediation for theft response, tenant determinism, and secret-only refresh hashing.

## Corrective Findings and Implemented Remediation

1. Finding (H01): refresh-token reuse did not revoke attacker successor/family.
   - Implemented remediation:
     - Added replay incident response in `backend/app/services/auth_tokens.py`.
     - On replay of rotated/revoked token with valid secret, revoke tenant+user refresh session set and emit `auth_refresh_family_revoked_on_reuse`.
     - Adjusted refresh transaction boundaries in `backend/app/api/auth.py` so revocation commits before opaque 401.

2. Finding (H02): tenant binding at issuance was not contract-mandatory.
   - Implemented remediation:
     - Updated `api-contracts/openapi/v1/auth.yaml` to require `tenant_id` for `POST /api/auth/login`.
     - Re-bundled contract to `api-contracts/dist/openapi/v1/auth.bundled.yaml`.
     - Updated `backend/app/schemas/auth.py` login request model to required `tenant_id`.
     - Added contract test `tests/contract/test_contract_semantics.py::test_auth_login_contract_requires_tenant_id`.

3. Finding (H03): refresh hashing boundary used full token blob, not secret-only.
   - Implemented remediation:
     - Refactored hashing in `backend/app/services/auth_tokens.py` to `bcrypt(sha256(secret))` for storage and verify.
     - Kept selector components (`tenant_id`, `token_id`) plaintext for O(1) lookup and clear selector/verifier boundary.
     - Added mutation hook `SKELDIR_B12_P4_HASH_BLOB=1` for negative-control proof.

4. Finding (H04): attacker-first theft scenario was missing from tests.
   - Implemented remediation:
     - Added `backend/tests/test_b12_p4_token_lifecycle.py::test_refresh_reuse_revokes_family_and_kills_successor`.
     - Added supporting gate tests for tenant determinism and secret-only hashing.
