## Section 3 - Actual System Map

### Provider/integration surfaces
- Platform substrate exists via `/api/attribution/platform-connections` and `/api/attribution/platform-credentials`.
- Provider runtime abstraction exists (`ProviderRegistry`, `StripeRevenueProvider`, `DummyRevenueProvider`) and is consumed by realtime revenue endpoints.
- Webhook ingestion uses provider-specific signature verification and tenant API key auth.
- No provider OAuth handshake routes (authorize/callback/disconnect/status lifecycle).

### Auth boundaries
- JWT user auth: `accessBearerAuth` + `get_auth_context` + scope checks.
- Webhook auth: `tenantKeyAuth` + HMAC verification per provider.
- Error normalization: 401/403/auth header failures normalized to canonical ProblemDetails.

### Tenant boundary enforcement
- HTTP plane injects tenant/user from JWT or webhook tenant key.
- DB plane enforces `tenant_id = current_setting('app.current_tenant_id')::uuid` with FORCE RLS on provider tables.
- Worker plane enforces authority envelope header for tenant-scoped tasks (`TenantTask`), with tenant/user context propagation and GUC application.

### Secret retrieval path
- Single choke point in `app/core/secrets.py`.
- Platform crypto material resolved via key ring (`get_platform_encryption_material_for_write`, `resolve_platform_encryption_key_by_id`).
- Provider token storage and webhook secret resolution both consume this substrate.

### Persistence path
- Provider records:
  - `platform_connections`: tenant-scoped connection identity/status/metadata.
  - `platform_credentials`: encrypted access/refresh tokens + `expires_at` + `scope` + `token_type` + `key_id`.
- Writes use `pgp_sym_encrypt`; reads decrypt in DB via `pgp_sym_decrypt` only when needed.

### Worker path
- Celery on Postgres-only broker/backend (`sqla+postgresql://`, `db+postgresql://`).
- Bounded controls active: late ack, time limits, prefetch=1, retry/backoff settings.
- Beat schedule exists for matview pulse, denylist GC, PII scan, retention; no provider token refresh schedule.

### Error path
- Canonical `application/problem+json` via `problem_details_response`.
- Auth failures converge to stable codes (`AUTH_UNAUTHORIZED`, `AUTH_FORBIDDEN`).
- Provider refresh/unavailability currently collapsed to generic 503 contract surface.

### CI adjudication path
- Required checks are enforced via governance contract + CI guardrail script.
- Branch protection currently requires B1.2 checks and multiple legacy checks, but no B1.3-specific required checks.
