# B1.2-P4 EG2a/EG5a Baseline

Date: 2026-03-03
Branch baseline audited: `main` (local fast-forward to `origin/main`)

## H01 - Token-id miss path timing oracle exists

- Verdict: **Validated**.
- Evidence:
  - `backend/app/services/auth_tokens.py:278` returns immediately when `token_row is None`.
  - No miss-path `bcrypt.checkpw` call before return.
- Impact:
  - Lookup miss has materially lower cost than lookup-hit+verify path, enabling selector (`token_id`) timing differential.

## H02 - Revocation scope is over-broad (user+tenant, not lineage)

- Verdict: **Validated**.
- Evidence:
  - `revoke_refresh_family` query filters only by `tenant_id` and `user_id` at `backend/app/services/auth_tokens.py:185-186`.
  - No `family_id` predicate is present in that query.
- Impact:
  - Reuse-triggered revocation can revoke all refresh families for same user+tenant (cross-device/session blast radius).

## H03 - No proof that parallel family survives reuse-triggered revocation

- Verdict: **Validated**.
- Evidence:
  - Existing attacker-first test `backend/tests/test_b12_p4_token_lifecycle.py:451` proves successor in same family dies.
  - No test exists that creates two families for same `(tenant_id,user_id)` and proves family-B remains usable after family-A reuse incident.
- Impact:
  - CI can pass while destructive cross-family revocation remains undetected.

## H04 - No EG2a negative control for miss-path dummy-bcrypt bypass

- Verdict: **Validated**.
- Evidence:
  - Negative-control harness has no EG2a mutation env (`SKELDIR_B12_P4_DISABLE_MISS_DUMMY_BCRYPT` absent) in `scripts/contracts/run_negative_controls.sh`.
  - Existing P4 negatives only include rotation bypass, plaintext, family-reuse bypass, hash-blob, and tenant-required contract drift (`scripts/contracts/run_negative_controls.sh:377-399`).
- Impact:
  - Miss-path constant-time hardening could regress silently.

## H05 - Current evidence language implies non-lineage revocation intent

- Verdict: **Validated** (wording consistent with code behavior).
- Evidence:
  - Evidence doc text states revoking tenant+user session set (aligned with current predicate behavior in H02).
  - Code path confirms this behavior (`backend/app/services/auth_tokens.py:185-186`).
- Impact:
  - Documentation and implementation currently align to over-broad revocation, not lineage-only semantics.

EG2a and EG5a are not satisfied on current `main` baseline:
