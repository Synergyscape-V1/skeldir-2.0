## Final Findings and Remediation Applied

### Findings observed during closure validation

1. Schema parity gates (`EG2.1` path and `validate-schema-authority`) failed on non-semantic RLS policy role-order drift in canonical SQL.
2. Migration execution under least-privilege runtime surfaced FK privilege gaps for `auth_refresh_tokens -> users`.
3. Secret chokepoint gate flagged direct secret settings access in auth code.
4. Static/runtime security scans flagged forbidden direct `os.environ["DATABASE_URL"]` access pattern in P4 test module.
5. Frontend contract-consumption gate reported generated `auth.ts` drift.

### Remediation implemented

1. Canonical schema authority fix: normalized RLS policy role ordering in `db/schema/canonical_schema.sql` to match migration-derived output exactly.
2. Migration privilege closure: added explicit `REFERENCES` grants for `public.users` in `alembic/versions/007_skeldir_foundation/202603021130_b12_p4_token_refresh_lifecycle.py` (and corresponding downgrade revokes).
3. Secret boundary closure: replaced direct settings secret read in `backend/app/api/auth.py` with sanctioned `get_secret(...)` path and in-process cache helper.
4. Scan-safe test refactor: replaced direct env-indexing in `backend/tests/test_b12_p4_token_lifecycle.py` with helper-based lookup compatible with static gate policy.
5. Contract consumption closure: regenerated `frontend/src/types/api/auth.ts` from bundled auth contract.
6. Canonical schema refreshed from normalized migration-derived schema and validated with schema authority script.
