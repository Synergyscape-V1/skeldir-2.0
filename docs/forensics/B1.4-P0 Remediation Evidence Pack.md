## 3. Privacy Authority Model (P0 Canonical)

Canonical artifact: `contracts-internal/governance/b14_p0_privacy_authority.main.json`

Model adjudications:
1. Banned direct identifier keys are explicitly enumerated.
2. Banned proxy identity keys are explicitly enumerated.
3. Durable immutable payloads are constrained by allowlist + forbidden raw-envelope keys.
4. Session boundary is explicitly privacy-grade (`uuid4_nondeterministic`, max 30 minutes contract).
5. Expirable surfaces are explicitly mapped (`dead_events`, `dead_events_quarantine` raw payloads).
6. Deletion contract is explicitly modeled as internal control-plane/worker authority.
7. Export allowlist and log/artifact no-leak key rules are machine-defined.

## 4. Event Lifecycle Resolution Chosen

Chosen architecture:
1. `attribution_events` remains immutable append-only durable substrate.
2. Durable payload in immutable events is minimized at ingestion and protected by insert-time DB trigger guard.
3. Mutable dead-letter raw envelopes are privacy-expirable via retention redaction.
4. Mutable dead-letter/quarantine surfaces are authority-erasable through internal worker path.

This resolves P0 contradiction by distinguishing durable minimal facts from expirable/erasable mutable envelopes.

## 5. Deletion Contract Exposure Decision

Decision:
1. Public DSAR endpoint shape is **not** introduced in P0.
2. P0 deletion exposure is **internal authority-controlled worker path**:
   - `app.tasks.privacy.erase_tenant_privacy_surfaces`
3. Erasure scope in P0:
   - `dead_events.raw_payload`
   - `dead_events.error_detail`
   - `dead_events_quarantine.raw_payload`
   - `dead_events_quarantine.error_detail`
4. Immutable `attribution_events` is explicitly non-erasable surface in this phase model.
