## 4) Root-Cause Findings

1. P0-P6 intentionally staged substrates ahead of orchestration, leaving refresh runtime composition deferred.
2. Token freshness responsibility was fragmented across call-sites without one canonical resolver.
3. Worker infrastructure existed, but refresh-specific scheduling, claiming, and locking policy was absent.
4. Terminal provider invalidation semantics were not encoded distinctly from transient failure policy.
5. Secret-boundary hardening existed at ingress/log layers but was not fully closed across task failure persistence surfaces.
