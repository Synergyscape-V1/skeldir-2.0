# B1.2-P3 v4 Corrective Evidence

Date: 2026-03-02  
Scope: Postgres-only P3 closure for transaction lock physics, bounded lock-miss behavior, hot-path DB elimination proof, and required-check enforcement.

## 1) Initial findings
- Baseline report: `docs/forensics/B1.2-P3_V4_BASELINE.md`
- Pre-fix verifier refresh used session advisory lock semantics and explicit unlock path.
- Lock-miss behavior for unknown-`kid` lacked bounded polling.
- No explicit CI proof that hot-path verification performs zero DB queries per request/task.

## 2) Remediations implemented
- `backend/app/core/secrets.py`
  - Replaced session lock usage with transaction-scoped try-lock:
    - `pg_try_advisory_xact_lock(...)`
  - Removed session unlock pattern from verifier refresh path.
  - Added `SET LOCAL statement_timeout` in refresh transaction path.
  - Added bounded lock-miss polling for unknown-`kid`:
    - bounded wait budget (default 150ms; clamped 50-250ms)
    - exponential polling backoff with jitter
    - cache-version observation using `fetched_at`
    - bounded miss returns to fail-closed JWT path (401 via invalid token surface).
  - Added test-only negative-control toggle for forced DB reads on verify:
    - `SKELDIR_B12_P3_FORCE_DB_READ_ON_VERIFY`
- `backend/tests/test_b12_p3_jwt_verification_standardization.py`
  - Added explicit hot-path 0-DB tests (API shared verifier, worker, HTTP).
- `backend/tests/test_b12_p3_v4_lock_semantics.py`
  - Added direct tests for:
    - xact lock SQL usage (`pg_try_advisory_xact_lock`)
    - lock-miss bounded polling success on cache update
    - lock-miss bounded polling timeout path.
- `.github/workflows/ci.yml`
  - Added `backend/tests/test_b12_p3_v4_lock_semantics.py` to required job `JWT Tenant Context Invariants`.
- `scripts/contracts/run_negative_controls.sh`
  - Added non-vacuous negative control:
    - `SKELDIR_B12_P3_FORCE_DB_READ_ON_VERIFY=1` must fail 0-DB hot-path proof.
