# B1.2-P5 GC Singleflight Corrective Evidence

## Summary
- Added Postgres transaction-scoped advisory lock singleflight to denylist GC.
- Added deterministic concurrent-GC proof test.
- Added non-vacuous negative control toggle to force concurrency proof failure when singleflight is disabled.

## Code Changes

### GC singleflight lock
- File: `backend/app/tasks/maintenance.py`
- Function: `_delete_expired_denylist_rows(batch_size)`
- Added:
  - lock key constant `_DENYLIST_GC_SINGLEFLIGHT_LOCK_KEY = 1205001`
  - `SELECT pg_try_advisory_xact_lock(:lock_key)` before any delete
  - lock miss behavior: clean no-op return (`deleted_rows=0`, `lock_acquired=0`)
  - mutation toggle: `SKELDIR_B12_P5_DISABLE_GC_SINGLEFLIGHT=1` (test-only negative control path)

### Deterministic concurrency proof
- File: `backend/tests/test_b12_p5_revocation_substrate.py`
- Test:
  - `test_denylist_gc_singleflight_allows_only_one_active_deleter`
- Proof behavior:
  - seeds expired denylist rows
  - runs two `_delete_expired_denylist_rows(...)` invocations concurrently
  - asserts lock outcomes are exactly `[0, 1]`
  - asserts deleter made progress and lock-miss path performed no delete
- Determinism aid:
  - uses `SKELDIR_B12_P5_GC_SINGLEFLIGHT_HOLD_SECONDS` to force overlap window in test context.

### Negative control wiring
- File: `scripts/contracts/run_negative_controls.sh`
- Added gate:
  - `SKELDIR_B12_P5_DISABLE_GC_SINGLEFLIGHT=1`
  - runs `test_denylist_gc_singleflight_allows_only_one_active_deleter`
  - expected failure; if it passes, script exits non-zero.
