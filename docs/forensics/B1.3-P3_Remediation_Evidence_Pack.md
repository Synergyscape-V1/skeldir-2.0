## Root-Cause Findings

1. Provider token encryption substrate existed, but durable lifecycle truth was underspecified for refresh/degradation targeting.
2. Queryability for scheduler-critical state required explicit hardening (typed columns + indexes + runtime plan proof).
3. Without a dedicated P3 gate, branch-protection/CI adjudication could remain advisory; governance contract and workflow context were extended.

## Exact Remediations Performed

1. Added in-place durable lifecycle migration for `platform_credentials`:
   - `next_refresh_due_at`, `lifecycle_status`, `refresh_failure_count`, `last_failure_class`, `last_failure_at`, `last_refresh_at`, `revoked_at`.
   - lifecycle constraints and due/revocation indexes.
   - explicit RLS/FORCE RLS + tenant policy reassertion for modified durable table.
2. Extended `PlatformCredential` ORM with lifecycle fields, checks, and indexes.
3. Extended `PlatformCredentialStore` with durable transition operations:
   - `list_refresh_due`, `record_refresh_failure`, `mark_refresh_success`, `mark_revoked`.
4. Extended `PlatformCredentialService.get_credentials` and typed response with lifecycle metadata.
5. Updated platform credential test builder to support new lifecycle columns.
6. Added structural enforcement script `scripts/ci/enforce_b13_p3_durable_lifecycle.py`.
7. Added non-vacuous structural tests (`backend/tests/test_b13_p3_durable_lifecycle_contract.py`).
8. Added runtime proofs (`backend/tests/test_b13_p3_durable_lifecycle_runtime.py`) covering:
   - durable state transitions,
   - metadata/ciphertext type separation,
   - indexed due-selection EXPLAIN without decryption,
   - RLS continuity on `platform_connections` and `platform_credentials`.
9. Added independent CI job `B1.3 P3 Durable Lifecycle Schema Proofs` in `.github/workflows/ci.yml`.
10. Added required context to governance contract `contracts-internal/governance/b03_phase2_required_status_checks.main.json`.
11. Updated canonical schema authority in `db/schema/canonical_schema.sql` for the P3 extension.
