# B1.2-P5 Corrective Evidence (Hot-Path I/O Elimination + Denylist GC)

## Change Set

### 1) Revocation cache + event propagation
- Added `backend/app/security/revocation_runtime.py`.
- Implements process-local bounded cache:
  - denylisted JTIs keyed by `(tenant_id,user_id,jti)` with TTL eviction.
  - kill-switch cutoffs keyed by `(tenant_id,user_id)`.
  - clean-token memoization keyed by `(tenant_id,user_id,jti)` with `exp` eviction.
- Implements Postgres `LISTEN/NOTIFY` listener channels:
  - `skeldir_token_revoked`
  - `skeldir_tokens_invalid_before_updated`

### 2) Canonical verifier moved to cache-first hot path
- Updated `backend/app/security/auth.py`:
  - `assert_access_token_active(...)` now checks cache first.
  - cold-start fallback uses DB only when cache state is unknown.
  - steady-state path avoids per-request DB revocation queries.
  - mutation toggle for negative control:
    - `SKELDIR_B12_P5_FORCE_DB_REVOCATION_HOT_PATH=1`

### 3) Revocation writes emit NOTIFY payloads
- Updated `backend/app/services/auth_revocation.py`:
  - `denylist_access_token(...)` emits `pg_notify` with `{tenant_id,user_id,jti,expires_at}`.
  - `upsert_tokens_invalid_before(...)` emits `pg_notify` with `{tenant_id,user_id,tokens_invalid_before}`.
  - evaluation now returns substrate fields needed for cache priming on cold path:
    - `denylist_expires_at`
    - `tokens_invalid_before`

### 4) Denylist GC implementation + schedule
- Updated `backend/app/tasks/maintenance.py`:
  - Added `_delete_expired_denylist_rows(batch_size)` with bounded batched deletes.
  - Added `gc_expired_access_token_denylist` Celery task.
  - mutation toggle for negative control:
    - `SKELDIR_B12_P5_GC_UNBOUNDED_DELETE=1`
- Updated `backend/app/tasks/beat_schedule.py`:
  - Added scheduled job `auth-denylist-gc` (every 10 minutes).
  - mutation toggle for negative control:
    - `SKELDIR_B12_P5_DISABLE_DENYLIST_GC_JOB=1`

### 5) CI negative controls expanded
- Updated `scripts/contracts/run_negative_controls.sh`:
  - force DB hot-path mutation must fail hot-path test.
  - disable listener mutation must fail propagation SLA test.
  - unbounded GC mutation must fail bounded GC test.
  - GC beat job removal mutation must fail schedule gate test.
